Tab 1: The tab itself will be titled "Text" and the contents need to be a long page of text where 
the user will need to scroll down to be able to see all of it. An example of how Tab 1 might appear. 
Notice how the text has a scrollbar but not the navigation bar with the tabs.

Tab 2: The tab itself will be titled "LR" and the contents will be the text "Down Here" positioned 
in the lower right corner of the app's area. This HTML example might have useful ideas.  An example 
of how Tab 2 might appear.
https://www.cs.umd.edu/class/spring2026/cmsc434/P1_2/Tab2.png 

Tab 3: The tab itself will be titled "Colors" and the contents will be colorful - specifically two 
lines of text where the first line has a text color of yellow and a background of black, and the 
second line has a text color of purple and a background color of yellow, followed by a pie chart 
which uses multiple bright colors.  This HTML example might have useful ideas if you want to have 
the pie chart generated dynamically.  An example of how Tab 3 might appear.
https://www.cs.umd.edu/class/spring2026/cmsc434/P1_2/Tab3.png

centered bold name under the image, such that when the user taps the image a notification appears. 
The user should then be able to close the notification.  This HTML example might have useful ideas.  
An example of how Tab 4 might appear at first and an example of how Tab 4 might appear after tapping the image.

Tab 5: The tab itself will be titled "Choices" and the contents need to include (1) something 
with a radio button entry option, (2) something with a drop-down list entry option, (3) a 
traditional button that when pressed will display text that is somehow reflecting the options chosen. 
There are various ways to accomplish this, such as in HTML.  An example of how Tab5 might appear.
https://www.cs.umd.edu/class/spring2026/cmsc434/P1_2/Tab5.png 

Tab 6: The tab itself will be titled "ToDo" and the contents need to be a functional, basic, 
"To Do" list where the user can type in a new item and add it to the list, and the user can 
then either delete things from the list or be able to "cross off" things on the list in some 
meaningful manner. There are various ways to accomplish this, such as in HTML.  An example of 
how Tab 6 might appear.
